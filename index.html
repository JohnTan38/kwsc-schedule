🔥 Firebase Firestore Setup Guide
Follow these steps once to connect this app to Google Firebase. After setup, all registration data is stored in the cloud — accessible from any device, any browser, even after the computer is turned off. Estimated time: 10–15 minutes.
1
Create a free Google Firebase project
Sign in to Google and create a new Firebase project — no credit card needed
▾

Go to 🔗 console.firebase.google.com ↗

Sign in with a Google account (use the KWSC admin Google account if you have one).

Click Add project → name it kwsc-mcnair → disable Google Analytics (not needed) → click Create project.

✅Firebase free tier (Spark plan) gives you 1 GB storage, 50,000 reads/day, 20,000 writes/day — more than enough for KWSC.
2
Create a Firestore Database
Set up the cloud database inside your project
▾

Inside your new project, click Build → Firestore Database in the left sidebar.

Click Create database. When asked about rules, choose Start in production mode (we will set proper rules in Step 4).

Choose location: asia-southeast1 (Singapore) — closest to KWSC, fastest response time.

ℹ️The database location cannot be changed later. Always pick asia-southeast1 for Singapore-based apps.
3
Enable Anonymous Authentication
Required: lets users save registrations without creating accounts
▾

In the left sidebar click Build → Authentication → click Get started.

Under the Sign-in method tab, click Anonymous → toggle it to Enabled → click Save.

ℹ️Why Anonymous Auth? Firestore security rules require a logged-in user before writing data. Anonymous Auth silently signs in each visitor without usernames/passwords — the user never sees a login screen, but Firebase knows the write is coming from a legitimate browser session, not a bot.
⚠️Without Anonymous Auth enabled, all registration submissions will fail with a permission-denied error.
4
Set Firestore Security Rules
Allow public writes (registration), restrict reads to authenticated users only
▾

In Firestore, click the Rules tab at the top. Replace all existing content with the rules below, then click Publish.

📋 Firestore Security Rules — copy exactly
rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { // REGISTRATIONS collection // Anyone signed in (including anonymous) can create a registration // Only authenticated users can read (protects personal data) match /registrations/{docId} { allow create: if request.auth != null; allow read, update, delete: if request.auth != null; } } }
✅These rules mean: any browser that has done Anonymous Auth (automatic) can write registrations. Reads and deletes require auth too — so the data is never publicly exposed.
5
Get your Firebase config keys
Copy the project credentials so this app can connect
▾

In the Firebase console, click the ⚙️ gear icon (top-left) → Project settings.

Scroll to Your apps → click the </> (Web) icon → name it kwsc-web → click Register app.

Firebase will show a config object like this:

const firebaseConfig = { apiKey: "AIzaSy...", authDomain: "kwsc-mcnair.firebaseapp.com", projectId: "kwsc-mcnair", storageBucket: "kwsc-mcnair.appspot.com", messagingSenderId: "123456789", appId: "1:123456789:web:abc123" };

Copy each value from that screen and paste it into the fields below.

⚠️The apiKey is not a secret — it is safe to include in a web page. It only identifies your project. The Security Rules (Step 4) are what protect your data.
6
Paste your config & connect
Enter your Firebase credentials below — saved permanently in this browser
▾
🔑 Firebase Configuration
💾Your config is saved in localStorage under key kwsc_fb_config. You only need to do this once per browser. If you deploy to Vercel, paste the config on each new device that needs admin access.
📥 How Admin Retrieves All Registration Data
Three ways to access every registration record from Firestore:
① This Dashboard (recommended) Unlock Admin tab with password → all registrations load live from Firestore → search, filter by week/activity → click Export CSV to download a spreadsheet-ready file.
② Firebase Console directly Go to console.firebase.google.com → your project → Firestore Database → registrations collection. Every document is one registration. Click any document to see fields. Use the export icon at the top to download JSON.
③ Firebase Admin SDK (advanced) For bulk exports, a Node.js script using the Firebase Admin SDK can query all documents with filtering, sorting, and export to CSV. Ask your developer if needed.
